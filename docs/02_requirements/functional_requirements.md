# 기능 요구사항 정의서

## 1. 문서 목적

이 문서는 StudyMate MVP에서 구현해야 하는 기능 요구사항을 기능 단위로 정의한다.

## 2. 핵심 내용

| ID | 기능 | 우선순위 | 설명 |
| --- | --- | --- | --- |
| FR-001 | 회원가입 | 필수 | 이메일과 비밀번호로 계정을 생성한다 |
| FR-002 | 로그인 | 필수 | 이메일과 비밀번호로 로그인한다 |
| FR-003 | 로그아웃 | 필수 | 현재 계정에서 로그아웃한다 |
| FR-004 | 학습 내용 입력 | 필수 | 제목, 과목명, 학습 내용을 입력한다 |
| FR-005 | AI 요약 생성 | 필수 | 입력한 학습 내용을 3~5줄 요약으로 변환한다 |
| FR-006 | AI 퀴즈 생성 | 필수 | 객관식 4지선다 퀴즈 3개를 생성한다 |
| FR-007 | 퀴즈 풀이 | 필수 | 사용자가 보기 중 하나를 선택하고 다음 문제로 이동한다 |
| FR-008 | 결과 확인 | 필수 | 정답 수, 전체 문제 수, 정답률을 표시한다 |
| FR-009 | 오답노트 저장 | 필수 | 틀린 문제를 자동 저장한다 |
| FR-010 | 학습 기록 조회 | 필수 | 사용자의 과거 학습 기록 목록을 표시한다 |
| FR-011 | 마이페이지 | 권장 | 사용자 이메일, 통계, 로그아웃 버튼을 표시한다 |
| FR-012 | PDF 텍스트 가져오기 | 권장 | PDF에서 텍스트를 추출해 학습 내용 입력칸에 반영한다 |
| FR-013 | 이미지 OCR | 권장 | 이미지에서 한국어 텍스트를 인식해 학습 내용 입력칸에 반영한다 |

## 3. 상세 설명

### FR-001 회원가입

- 사용자는 이메일과 비밀번호를 입력해 계정을 생성할 수 있다.
- 비밀번호 확인 값이 일치해야 한다.
- 회원가입 성공 시 users 컬렉션에 사용자 기본 정보를 저장한다.

### FR-002 로그인

- 사용자는 이메일과 비밀번호로 로그인할 수 있다.
- 로그인 성공 시 홈 화면으로 이동한다.
- 로그인 상태는 앱 재실행 후에도 유지한다.

### FR-004 학습 내용 입력

- 사용자는 제목, 과목명, 학습 내용을 입력한다.
- 사용자는 PDF 또는 이미지 OCR로 추출한 텍스트를 학습 내용으로 사용할 수 있다.
- 제목은 필수이다.
- 학습 내용은 최소 30자 이상이어야 한다.
- 너무 짧은 입력은 AI 요약 요청을 차단한다.

### FR-005 AI 요약 생성

- AI는 핵심 요약 3~5줄과 키워드 3개를 JSON 형식으로 반환한다.
- 앱은 JSON을 파싱해 요약 결과 화면에 표시한다.
- 요약 결과는 Firestore의 study_notes에 저장한다.

### FR-006 AI 퀴즈 생성

- AI는 객관식 4지선다 문제 3개를 생성한다.
- 각 문제는 question, options, answerIndex, explanation 필드를 가진다.
- 생성된 문제는 quizzes 컬렉션에 저장한다.

### FR-007 퀴즈 풀이

- 사용자는 문제별로 보기 1개를 선택한다.
- 다음 버튼으로 다음 문제로 이동한다.
- 마지막 문제 완료 후 결과 화면으로 이동한다.

### FR-009 오답노트 저장

- 사용자가 틀린 문제는 wrong_answers 컬렉션에 저장한다.
- 저장 항목에는 문제, 보기, 선택한 답, 정답, 해설이 포함된다.

## 4. 개발 시 참고사항

- 기능 구현 순서는 인증, 학습 기록, AI 요약, AI 퀴즈, 결과 저장, 오답노트 순서가 적절하다.
- AI 응답 파싱 실패는 사용자에게 명확한 오류 메시지로 안내한다.
- Firestore 조회는 반드시 현재 로그인 사용자 `userId` 기준으로 필터링한다.
- MVP에서는 퀴즈 수를 기본 3개로 고정한다.
