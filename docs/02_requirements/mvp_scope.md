# MVP 범위 정의서

## 1. 문서 목적

이 문서는 2개월 개발 기간 동안 StudyMate에서 반드시 구현할 기능과 제외할 기능을 명확히 구분한다.

## 2. 핵심 내용

### MVP 포함 기능

| 번호 | 기능 | 설명 |
| --- | --- | --- |
| 1 | 회원가입 / 로그인 | Firebase Authentication 기반 이메일 인증 |
| 2 | 학습 내용 입력 | 제목, 과목명, 학습 내용 입력 |
| 3 | AI 요약 생성 | 핵심 요약 3~5줄, 키워드 3개 생성 |
| 4 | AI 객관식 퀴즈 생성 | 4지선다 문제 3개 생성 |
| 5 | 퀴즈 풀이 | 답 선택 및 다음 문제 이동 |
| 6 | 결과 확인 | 정답 수, 전체 문제 수, 정답률 표시 |
| 7 | 오답노트 저장 | 틀린 문제 자동 저장 |
| 8 | 학습 기록 목록 | 과거 학습 기록 조회 |

### MVP 제외 기능

- PDF 자동 분석
- 이미지 OCR 분석
- 음성 녹음 요약
- 실시간 채팅형 AI 튜터
- 친구 공유 기능
- 랭킹 시스템
- 유료 결제
- 복잡한 관리자 페이지

## 3. 상세 설명

StudyMate MVP의 핵심 흐름은 `입력 -> 요약 -> 퀴즈 -> 결과 -> 오답`이다. 이 흐름을 벗어나는 기능은 2개월 개발 범위에서 제외한다.

MVP에서 가장 중요한 검증 목표는 다음과 같다.

- 사용자가 학습 내용을 입력할 수 있는가?
- AI가 요약을 반환하고 앱이 이를 표시할 수 있는가?
- AI가 퀴즈 JSON을 반환하고 앱이 이를 파싱할 수 있는가?
- 사용자가 퀴즈를 풀고 결과를 확인할 수 있는가?
- 틀린 문제를 오답노트로 다시 볼 수 있는가?

제외 기능은 향후 확장 가능하지만, 최초 구현 단계에서는 개발 일정과 안정성을 위해 포함하지 않는다.

## 4. 개발 시 참고사항

- AI 퀴즈 문제 수는 기본 3개로 고정한다.
- Storage는 PDF, 이미지, 음성 업로드 기능이 제외되므로 MVP 필수 의존성이 아니다.
- 친구 공유, 랭킹, 결제는 사용자 모델과 DB 구조를 복잡하게 만들기 때문에 제외한다.
- 관리자 페이지는 과제 제출용 MVP에서 우선순위가 낮다.
