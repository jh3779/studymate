# 발표 자료 구성안

## 1. 문서 목적

이 문서는 StudyMate 프로젝트 발표 자료를 만들 때 사용할 슬라이드 구성과 핵심 메시지를 정리한다.

## 2. 핵심 내용

| 순서 | 슬라이드 제목 | 핵심 메시지 |
| --- | --- | --- |
| 1 | 프로젝트 소개 | StudyMate는 AI 기반 학습 요약 및 퀴즈 생성 앱이다 |
| 2 | 문제 정의 | 학습자는 요약과 문제 제작에 많은 시간을 쓴다 |
| 3 | 해결 방안 | AI가 요약과 퀴즈를 자동 생성한다 |
| 4 | 대상 사용자 | 학생과 시험 대비 학습자가 주요 사용자다 |
| 5 | 핵심 기능 | 로그인, 학습 입력, AI 요약, 퀴즈, 결과, 오답노트 |
| 6 | 화면 흐름 | 입력부터 오답 복습까지 이어지는 모바일 플로우 |
| 7 | 시스템 구조 | Android Java, Firebase, AI API 기반 구조 |
| 8 | DB 설계 | users, study_notes, quizzes, quiz_results, wrong_answers |
| 9 | AI 프롬프트 | JSON 기반 요약 및 퀴즈 응답 설계 |
| 10 | 개발 일정 | 8주 MVP 개발 계획 |
| 11 | 시연 | 학습 입력부터 오답노트까지 데모 |
| 12 | 기대 효과 | 복습 시간 단축과 문제 풀이 중심 학습 지원 |

## 3. 상세 설명

### 발표 핵심 흐름

1. 학습자가 겪는 문제를 먼저 제시한다.
2. StudyMate가 어떤 방식으로 문제를 해결하는지 설명한다.
3. MVP 기능 범위를 명확히 소개한다.
4. 화면 흐름과 데이터 구조를 보여준다.
5. AI 프롬프트가 어떤 JSON을 반환하는지 설명한다.
6. 시연으로 전체 플로우를 확인시킨다.
7. 2개월 개발 일정과 확장 가능성을 정리한다.

### 강조할 내용

- 구현 범위를 현실적으로 제한했다.
- AI API Key 보안 리스크를 인지하고 문서화했다.
- Firestore 구조가 사용자별 학습 기록 관리에 맞춰져 있다.
- 오답노트를 통해 단순 요약 앱이 아니라 복습 앱으로 기능한다.

### 발표에서 제외하거나 축소할 내용

- PDF/OCR은 텍스트 입력 보조 기능으로 소개하고, 음성 요약 같은 확장 기능은 MVP 제외로만 언급한다.
- AI 모델별 세부 성능 비교는 이번 발표의 핵심이 아니다.
- 결제, 친구 공유, 랭킹은 구현 범위 밖이다.

## 4. 개발 시 참고사항

- 발표 자료에는 실제 API Key나 민감 정보가 보이지 않도록 한다.
- 시연 데이터는 짧고 명확한 학습 내용으로 준비한다.
- AI API 요청 실패 상황을 대비해 오류 안내와 재시도 절차를 준비한다.
- 발표 시간에 맞춰 데모는 3~5분 내로 구성한다.

- [ ] 보안 리스크가 언급되는가?
- [ ] 발표 시간이 과도하게 길지 않은가?
