# 시연 시나리오

## 1. 문서 목적

이 문서는 StudyMate 발표 또는 과제 제출 시 사용할 앱 시연 흐름과 준비 데이터를 정리한다.

## 2. 핵심 내용

시연 목표는 사용자가 학습 내용을 입력한 뒤 AI 요약, 퀴즈 생성, 풀이, 결과 확인, 오답노트 저장까지 이어지는 핵심 MVP 흐름을 보여주는 것이다.

## 3. 상세 설명

### 시연 준비

| 항목 | 내용 |
| --- | --- |
| 테스트 계정 | 발표용 이메일 계정 1개 |
| 학습 주제 | 데이터베이스 기본키 또는 한국사 주요 사건 등 짧은 주제 |
| 입력 길이 | 30자 이상, 1~2문단 수준 |
| 네트워크 | AI API 호출 가능한 환경 |
| 예비 자료 | API 실패 시 보여줄 오류 안내와 재시도 절차 |

### 예시 입력 데이터

```text
제목: 데이터베이스 기본키 정리
과목명: 데이터베이스
학습 내용:
기본키는 데이터베이스 테이블에서 각 행을 고유하게 식별하기 위해 사용하는 속성이다. 기본키 값은 중복될 수 없으며 NULL 값을 가질 수 없다. 하나의 테이블에는 기본키가 하나만 존재하며, 기본키는 개체 무결성을 유지하는 데 중요한 역할을 한다.
```

### 시연 순서

1. 앱 실행 후 SplashScreen에서 로그인 상태를 확인한다.
2. 로그인 화면에서 발표용 계정으로 로그인한다.
3. HomeScreen에서 오늘의 학습 시작 버튼을 누른다.
4. StudyInputScreen에 제목, 과목명, 학습 내용을 입력한다.
5. AI 요약 생성 버튼을 누른다.
6. SummaryResultScreen에서 요약 3~5줄과 키워드 3개를 확인한다.
7. 퀴즈 생성 버튼을 누른다.
8. QuizScreen에서 객관식 문제 3개를 순서대로 푼다.
9. QuizResultScreen에서 전체 문제 수, 맞힌 문제 수, 정답률을 확인한다.
10. 틀린 문제 보기 버튼을 눌러 WrongAnswerScreen으로 이동한다.
11. 오답 문제의 정답과 해설을 확인한다.
12. HomeScreen으로 돌아와 최근 학습 기록이 표시되는지 확인한다.

### 성공 기준

- 로그인 후 홈 화면으로 이동한다.
- 학습 내용 입력값 검증이 정상 동작한다.
- AI 요약 결과가 화면에 표시된다.
- AI 퀴즈 3개가 생성된다.
- 퀴즈 풀이 후 정답률이 계산된다.
- 틀린 문제가 오답노트에 저장된다.
- 학습 기록 목록에서 새 기록을 확인할 수 있다.

### 실패 대비 시나리오

- AI API 응답이 실패하면 로딩이 종료되고 오류 메시지를 표시한다.
- JSON 파싱 실패 시 재시도 안내를 표시한다.
- 네트워크 문제가 발생하면 오류 안내 화면을 보여주고, 연결 복구 후 같은 입력으로 다시 시도한다.

## 4. 개발 시 참고사항

- 발표 전 테스트 계정과 Firebase 데이터는 미리 준비한다.
- 시연 중 API 비용이 발생할 수 있으므로 반복 호출을 최소화한다.
- 오답노트 저장을 보여주려면 일부러 한 문제를 틀리게 선택한다.
- 앱 화면에 API Key나 디버그 로그가 노출되지 않도록 한다.
