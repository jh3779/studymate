# 개발 일정표

## 1. 문서 목적

이 문서는 StudyMate MVP를 약 2개월 동안 개발하기 위한 주차별 작업 계획을 정의한다.

## 2. 핵심 내용

주차별 작업은 [팀 작업 분담표](team_task_distribution.md)의 4개 파트 기준으로 병렬 진행한다.

| 주차 | 주요 목표 | 산출물 |
| --- | --- | --- |
| 1주차 | 기획 및 설계 | 주제 확정, 기능 목록, 화면 흐름도, DB 설계 |
| 2주차 | 프로젝트 기반 구축 | Flutter 프로젝트, Firebase 연동, 인증 화면 |
| 3주차 | 학습 기록 기반 기능 | 홈, 학습 입력, Firestore 저장 및 조회 |
| 4주차 | AI 요약 기능 | AIService 요약, 요약 결과 화면, 저장 |
| 5주차 | AI 퀴즈 기능 | 퀴즈 프롬프트, JSON 파싱, 퀴즈 화면 |
| 6주차 | 퀴즈 결과 기능 | 답안 선택, 점수 계산, 결과 저장, 오답 추출 |
| 7주차 | 오답노트 및 마이페이지 | 오답 목록, 상세, 다시 풀기, 로그아웃 |
| 8주차 | 마감 | 전체 테스트, 오류 수정, 발표 자료, 최종 보고서 |

## 3. 상세 설명

### 1주차

- 주제 확정
- 기능 목록 작성
- 화면 흐름도 작성
- DB 구조 설계
- Firebase 프로젝트 생성

### 2주차

- Flutter 프로젝트 생성
- Firebase 연동
- 폴더 구조 세팅
- 공통 위젯 생성
- 로그인 / 회원가입 화면 구현

### 3주차

- 홈 화면 구현
- 학습 입력 화면 구현
- Firestore 저장 구조 구현
- 학습 기록 목록 조회 구현

### 4주차

- AIService 구현
- 요약 프롬프트 구성
- API 요청 테스트
- 요약 결과 화면 구현
- 요약 데이터 저장

### 5주차

- 퀴즈 생성 프롬프트 구성
- AI 응답 JSON 파싱
- QuizModel 구현
- 퀴즈 데이터 저장
- 퀴즈 화면 구현

### 6주차

- 답안 선택 기능
- 점수 계산 기능
- 결과 화면 구현
- quiz_results 저장
- 틀린 문제 추출

### 7주차

- 오답노트 목록 구현
- 오답 상세 화면 구현
- 다시 풀기 기능 구현
- 마이페이지 구현
- 로그아웃 구현

### 8주차

- 전체 기능 테스트
- 오류 수정
- UI 정리
- 발표 자료 제작
- 시연 영상 준비
- 최종 보고서 작성

## 4. 개발 시 참고사항

- 4~5주차 AI 기능은 API 응답 불안정성과 파싱 오류 가능성이 있으므로 여유를 둔다.
- 6주차 결과 저장과 오답 저장은 Firestore 스키마와 정확히 맞춰 구현한다.
- 8주차에는 신규 기능을 추가하지 않고 안정화와 발표 준비에 집중한다.
- 일정 지연 시 오답 다시 풀기 또는 마이페이지 통계 세부 기능을 축소할 수 있다.

- [ ] 8주차 일정이 MVP 기능과 연결되어 있는가?
- [ ] AI 기능 구현과 테스트 기간이 확보되어 있는가?
- [ ] 마지막 주에 테스트와 발표 준비 시간이 있는가?
- [ ] 기능 추가보다 안정화 우선순위가 명확한가?
- [ ] 일정 지연 시 축소 가능한 기능이 있는가?
