# API 보안 유의사항

## 1. 문서 목적

이 문서는 StudyMate에서 AI API를 사용할 때 발생할 수 있는 보안 리스크와 MVP 단계의 처리 방안을 정리한다.

## 2. 핵심 내용

| 항목 | 권장 여부 | 설명 |
| --- | --- | --- |
| Android 앱에 API Key 하드코딩 | 금지 | 앱 패키지 분석으로 키가 노출될 수 있다 |
| Git 저장소에 API Key 커밋 | 금지 | 공개 또는 공유 저장소에서 즉시 노출된다 |
| 환경 변수 사용 | 개발용 허용 | 과제 MVP에서 제한적으로 사용 가능하다 |
| Cloud Functions 중계 | 권장 | 실제 배포 구조에서 API Key 보호에 적합하다 |
| 사용량 제한 설정 | 권장 | 키 유출 또는 오남용 피해를 줄인다 |

## 3. 상세 설명

### 주요 리스크

AI API Key를 Android 앱 내부에 직접 작성하면 빌드된 앱에서 키가 추출될 수 있다. 또한 Git 저장소에 `.env`, 설정 파일, 소스 코드 형태로 키가 커밋되면 저장소 접근 권한이 있는 모든 사람이 키를 사용할 수 있다.

AI API는 호출 비용이 발생할 수 있으므로 키가 노출되면 예상하지 못한 비용 문제가 발생할 수 있다.

### MVP 단계 방안

수업 과제 MVP에서는 실제 배포가 아니라 기능 검증이 목적이므로 다음 기준을 적용한다.

- API Key는 Git에 커밋하지 않는다.
- `.env` 또는 로컬 설정 파일을 사용한다면 `.gitignore`에 포함한다.
- README에는 API Key 입력 방법을 설명하되 실제 값을 작성하지 않는다.
- 발표 및 제출 자료에는 키를 노출하지 않는다.
- AI API 호출 실패 시 앱이 중단되지 않도록 예외 처리를 구현한다.

### 실제 배포 권장 구조

실제 배포를 고려한다면 앱이 AI API를 직접 호출하지 않고 서버 또는 Cloud Functions를 호출해야 한다.

```text
Android App
  -> Cloud Functions 또는 Backend Server
  -> AI API
```

이 구조에서는 API Key가 서버 환경 변수로 관리되며, 앱에는 직접 노출되지 않는다.

### 추가 보호 방안

- API 제공자 콘솔에서 사용량 제한을 설정한다.
- 가능한 경우 호출 도메인 또는 프로젝트 제한을 설정한다.
- 개발용 키와 배포용 키를 분리한다.
- 키가 노출되었다고 판단되면 즉시 폐기하고 재발급한다.

## 4. 개발 시 참고사항

- AIService 설계 문서에는 API Key 하드코딩 금지 주석을 포함한다.
- 과제 제출용 코드에는 예시 키나 실제 키를 포함하지 않는다.
- `.gitignore`에 환경 변수 파일을 포함해야 한다.
- Cloud Functions 구현은 MVP 필수 기능은 아니지만, 보안 권장 구조로 문서화한다.

- [ ] 수업 과제 MVP와 실제 배포 구조가 구분되어 있는가?
- [ ] Cloud Functions 중계 구조가 권장되어 있는가?
- [ ] 키 노출 시 대응 방법이 포함되어 있는가?
