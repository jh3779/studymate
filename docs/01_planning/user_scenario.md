# 사용자 시나리오

## 1. 문서 목적

이 문서는 StudyMate의 주요 사용자 유형과 실제 앱 사용 흐름을 시나리오 형태로 정리한다.

## 2. 핵심 내용

주요 사용자는 시험 대비 학습자이다.

- 대학생
- 고등학생
- 자격증 공부생
- 단기간 시험 대비를 하는 학습자

대표 사용 흐름은 학습 내용 입력, AI 요약 생성, AI 퀴즈 생성, 퀴즈 풀이, 결과 확인, 오답 복습 순서로 진행된다.

## 3. 상세 설명

### 기본 시나리오

1. 사용자가 앱을 실행한다.
2. 로그인 상태가 아니면 이메일로 로그인하거나 회원가입한다.
3. 홈 화면에서 오늘의 학습 시작 버튼을 누른다.
4. 제목, 과목명, 학습 내용을 입력한다.
5. AI 요약 생성 버튼을 누른다.
6. 앱이 입력값을 검증한 뒤 AI API에 요약을 요청한다.
7. 사용자는 요약 결과와 핵심 키워드를 확인한다.
8. 퀴즈 생성 버튼을 누른다.
9. 앱이 객관식 4지선다 문제 3개를 생성한다.
10. 사용자는 문제를 풀고 다음 문제로 이동한다.
11. 마지막 문제를 풀면 결과 화면에서 정답률을 확인한다.
12. 틀린 문제는 오답노트에 자동 저장된다.
13. 사용자는 홈 또는 오답노트에서 이전 학습 결과를 다시 확인한다.

### 예시 사용자

| 사용자 | 상황 | 기대 행동 |
| --- | --- | --- |
| 대학생 | 데이터베이스 수업 필기 복습 | 기본키, 정규화 같은 핵심 개념을 요약하고 퀴즈로 확인 |
| 고등학생 | 한국사 시험 대비 | 긴 교과서 내용을 붙여넣고 중요 사건 중심으로 복습 |
| 자격증 공부생 | 정보처리기사 이론 학습 | 암기 개념을 객관식 문제로 반복 확인 |

### 예외 시나리오

- 학습 내용이 30자 미만이면 요약 생성 버튼을 비활성화하거나 오류 메시지를 표시한다.
- AI API 요청이 실패하면 재시도 안내와 입력 내용 보존을 제공한다.
- 퀴즈 생성 결과가 JSON 형식에 맞지 않으면 사용자에게 오류를 알리고 다시 요청할 수 있게 한다.

## 4. 개발 시 참고사항

- 화면 이동은 `SplashScreen -> LoginScreen/HomeScreen -> StudyInputScreen -> SummaryResultScreen -> QuizScreen -> QuizResultScreen` 흐름을 기본으로 한다.
- 사용자가 작성한 원문은 AI 요청 실패 시에도 사라지지 않아야 한다.
- 오답노트는 틀린 문제만 저장하며, 다시 풀기 기능은 7주차 구현 범위로 둔다.
