# 프로젝트 제안서

## 1. 문서 목적

이 문서는 StudyMate 프로젝트를 수업 과제로 제안하기 위한 기획 배경, 목적, 기대 효과, 개발 범위를 정리한다.

## 2. 핵심 내용

| 구분 | 내용 |
| --- | --- |
| 제안 주제 | AI 기반 학습 요약 및 퀴즈 생성 앱 |
| 핵심 문제 | 학습자가 긴 자료를 직접 요약하고 문제화하는 데 많은 시간이 든다 |
| 해결 방식 | AI가 학습 내용을 요약하고 객관식 퀴즈를 자동 생성한다 |
| 기대 효과 | 복습 시간 단축, 문제 풀이 중심 학습, 오답 반복 학습 |
| 구현 범위 | 모바일 앱 MVP |

## 3. 상세 설명

StudyMate는 사용자가 공부한 내용을 앱에 입력하면 AI가 핵심 요약과 복습용 퀴즈를 제공하는 학습 보조 서비스이다. 학습자는 단순히 자료를 읽는 방식에서 벗어나, 요약된 핵심 내용을 확인하고 문제를 풀며 이해도를 점검할 수 있다.

프로젝트의 핵심 가치는 다음과 같다.

- 긴 학습 자료를 짧고 이해하기 쉬운 요약으로 변환한다.
- 요약 또는 원문을 기반으로 객관식 문제를 자동 생성한다.
- 퀴즈 결과와 오답노트를 통해 반복 학습을 지원한다.
- 사용자별 학습 기록을 저장해 이전 학습 내용을 다시 확인할 수 있다.

이번 프로젝트는 Android Native Java와 Firebase를 사용하여 빠르게 MVP를 구현하는 것을 목표로 한다. AI 기능은 OpenAI API 또는 Gemini API를 사용하며, API 응답은 JSON 형식으로 받아 앱에서 파싱한다.

## 4. 개발 시 참고사항

- 기능 확장보다 핵심 학습 플로우 완성에 집중한다.
- AI 응답 품질은 프롬프트 구조와 JSON 파싱 안정성에 영향을 받는다.
- Firebase Authentication을 사용해 사용자별 데이터를 분리한다.
- Firestore 문서 구조는 학습 기록, 퀴즈, 결과, 오답노트를 기준으로 설계한다.
