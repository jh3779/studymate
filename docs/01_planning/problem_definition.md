# 문제 정의서

## 1. 문서 목적

이 문서는 StudyMate가 해결하려는 학습 문제를 정의하고, 해당 문제가 왜 모바일 앱과 AI 기능으로 해결될 수 있는지 설명한다.

## 2. 핵심 내용

| 문제 | 설명 |
| --- | --- |
| 요약 부담 | 긴 학습 내용을 직접 요약하는 데 시간이 많이 든다 |
| 핵심 파악 어려움 | 어떤 내용이 중요한지 판단하기 어렵다 |
| 문제 제작 부담 | 복습 문제를 직접 만드는 과정이 번거롭다 |
| 반복 학습 부족 | 틀린 문제를 체계적으로 다시 보기 어렵다 |
| 기록 관리 부족 | 날짜별 학습 기록과 결과를 한곳에서 보기 어렵다 |

## 3. 상세 설명

시험 대비 학습자는 보통 강의자료, 필기, 교재 내용을 읽고 핵심 내용을 정리한다. 그러나 자료가 길수록 요약 시간이 늘어나며, 초보 학습자는 중요 개념을 놓치기 쉽다.

또한 복습 효과를 높이려면 단순 읽기보다 문제 풀이가 필요하지만, 문제를 직접 만드는 작업은 별도의 시간이 필요하다. 틀린 문제를 다시 정리하는 오답노트 역시 수작업으로 관리하면 누락되기 쉽다.

StudyMate는 다음 방식으로 문제를 해결한다.

- 사용자가 입력한 학습 내용을 AI가 핵심 요약으로 변환한다.
- 요약된 내용 또는 원문을 기반으로 객관식 문제를 생성한다.
- 퀴즈 풀이 결과를 저장하고 틀린 문제를 오답노트로 자동 분류한다.
- 학습 기록 목록에서 과거 학습 내용과 정답률을 확인한다.

## 4. 개발 시 참고사항

- 문제 해결의 중심은 AI 요약, AI 퀴즈, 오답 저장이다.
- AI가 생성한 내용은 항상 완전한 정답이라고 가정하지 않고, MVP에서는 사용자가 결과를 확인하는 학습 보조 도구로 정의한다.
- 입력값 검증을 통해 너무 짧은 텍스트로 AI 요청을 보내지 않도록 한다.
- 퀴즈 생성 개수는 기본 3개로 제한해 API 비용과 응답 시간을 관리한다.

## 5. 확인 체크리스트

- [ ] 사용자의 학습 불편이 구체적으로 정의되어 있는가?
- [ ] 앱 기능이 문제 해결 방식과 직접 연결되어 있는가?
- [ ] AI 기능이 과장되지 않고 보조 도구로 정의되어 있는가?
- [ ] 반복 학습과 기록 관리 문제가 포함되어 있는가?
- [ ] MVP 범위 안에서 해결 가능한 문제인가?
