# StudyMate 프로젝트 개요

## 1. 문서 목적

이 문서는 StudyMate 프로젝트의 전체 방향, 개발 범위, 기술 스택, 핵심 기능을 한눈에 확인하기 위한 최상위 설계 문서이다.

## 2. 핵심 내용

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | StudyMate |
| 프로젝트 유형 | AI 기반 학습 요약 및 퀴즈 생성 모바일 앱 |
| 개발 기간 | 약 2개월 |
| 주요 사용자 | 대학생, 고등학생, 자격증 공부생, 시험 대비 학습자 |
| Frontend | Android Native Java |
| Auth | Firebase Authentication |
| Database | Cloud Firestore |
| AI API | OpenAI API 또는 Gemini API |

한 줄 설명: 사용자가 입력한 학습 내용을 AI가 자동으로 요약하고, 객관식 퀴즈를 생성하여 복습할 수 있도록 돕는 모바일 학습 도우미 앱이다.

## 3. 상세 설명

학생들은 시험 공부 과정에서 긴 강의자료, 필기 내용, 교재 내용을 직접 요약하고 문제로 만들어야 한다. 이 과정은 시간이 오래 걸리며, 중요한 내용을 판단하는 데 어려움이 있다.

StudyMate는 사용자가 학습 내용을 텍스트로 입력하면 AI가 핵심 내용을 3~5줄로 요약하고, 요약 또는 원문을 기반으로 객관식 4지선다 퀴즈를 생성한다. 사용자는 생성된 문제를 풀고 정답률을 확인하며, 틀린 문제는 오답노트에 저장해 반복 학습할 수 있다.

이번 프로젝트는 수업 과제 제출과 실제 개발 착수를 모두 고려한다. 따라서 MVP 기능을 명확히 제한하고, 2개월 안에 구현 가능한 현실적인 기능만 포함한다.

## 4. 개발 시 참고사항

- PDF와 이미지 OCR은 기기에서 텍스트를 추출해 학습 내용 입력을 돕는 범위로 포함한다.
- 음성 요약, 친구 공유, 결제, 관리자 페이지는 MVP 범위에서 제외한다.
- AI API Key를 Android 앱 내부에 직접 하드코딩하지 않는다.
- 수업 과제 MVP에서는 개발용 환경 변수 또는 별도 설정 파일을 사용하되, 실제 배포 구조에서는 Cloud Functions 같은 서버 중계 구조를 권장한다.
- 사용자의 학습 데이터는 사용자별로 분리 조회되어야 한다.
- 문서 간 상세 내용은 각 하위 문서를 기준으로 관리한다.
