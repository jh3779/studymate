# 와이어프레임 생성 전달 프롬프트

## 목적

아래 프롬프트는 StudyMate의 현재 기획, 요구사항, 화면 설계 문서를 바탕으로 모바일 앱 와이어프레임을 생성하기 위해 사용한다.

## 전달 프롬프트

```text
너는 모바일 앱 UX/UI 디자이너다.
아래 제품 설명과 화면 요구사항을 바탕으로 StudyMate 모바일 앱의 저충실도 또는 중충실도 와이어프레임을 생성해라.

목표:
- Flutter로 구현할 수 있는 모바일 앱 화면 구조를 설계한다.
- MVP 핵심 흐름인 입력 -> 요약 -> 퀴즈 -> 결과 -> 오답 복습이 끊기지 않게 구성한다.
- 기능 확장보다 과제 제출 및 2개월 내 구현 가능한 MVP 완성도를 우선한다.
- 화면은 학습자가 빠르게 내용을 입력하고, AI 결과를 확인하고, 퀴즈를 풀 수 있도록 단순하고 직관적으로 설계한다.

제품 개요:
- 앱 이름: StudyMate
- 유형: AI 기반 학습 요약 및 퀴즈 생성 모바일 앱
- 주요 사용자: 대학생, 고등학생, 자격증 공부생, 단기간 시험 대비 학습자
- 핵심 가치:
  - 긴 학습 내용을 3~5줄 요약으로 변환
  - 핵심 키워드 3개 추출
  - 객관식 4지선다 퀴즈 3개 자동 생성
  - 퀴즈 결과와 정답률 확인
  - 틀린 문제를 오답노트로 자동 저장
  - 과거 학습 기록 조회

MVP 포함 기능:
- 이메일 회원가입
- 이메일 로그인
- 로그아웃
- 학습 내용 입력
- AI 요약 생성
- AI 객관식 퀴즈 생성
- 퀴즈 풀이
- 결과 확인
- 오답노트 저장 및 조회
- 학습 기록 목록
- 마이페이지

MVP 제외 기능:
- PDF 업로드
- 이미지 OCR
- 음성 녹음 요약
- 채팅형 AI 튜터
- 친구 공유
- 랭킹
- 결제
- 관리자 페이지

전체 화면 흐름:
SplashScreen
-> LoginScreen 또는 HomeScreen
-> SignUpScreen
-> HomeScreen
-> StudyInputScreen
-> SummaryResultScreen
-> QuizScreen
-> QuizResultScreen
-> WrongAnswerScreen
-> MyPageScreen

주요 사용자 흐름:
1. 사용자가 앱을 실행한다.
2. 로그인 상태를 확인한다.
3. 로그인하지 않은 경우 이메일로 로그인하거나 회원가입한다.
4. 홈 화면에서 오늘의 학습 시작을 누른다.
5. 제목, 과목명, 학습 내용을 입력한다.
6. AI 요약 생성 버튼을 누른다.
7. 요약 결과 3~5줄과 핵심 키워드 3개를 확인한다.
8. 퀴즈 생성 버튼을 누른다.
9. 객관식 4지선다 문제 3개를 순서대로 푼다.
10. 결과 화면에서 맞힌 문제 수, 전체 문제 수, 정답률을 확인한다.
11. 틀린 문제는 오답노트에 자동 저장된다.
12. 홈 또는 오답노트에서 이전 학습 결과를 다시 확인한다.

화면별 와이어프레임 요구사항:

1. SplashScreen
- 앱 로고 또는 StudyMate 텍스트
- 짧은 로딩 표시
- 로그인 상태 확인 중임을 암시하는 단순한 구조

2. LoginScreen
- 앱 이름
- 이메일 입력 필드
- 비밀번호 입력 필드
- 로그인 버튼
- 회원가입 화면 이동 버튼
- 로그인 실패 메시지 영역

3. SignUpScreen
- 이메일 입력 필드
- 비밀번호 입력 필드
- 비밀번호 확인 입력 필드
- 회원가입 버튼
- 로그인 화면으로 돌아가기 버튼
- 비밀번호 불일치 오류 메시지 영역

4. HomeScreen
- 상단 인사말 또는 사용자 학습 상태 요약
- 오늘의 학습 시작 버튼을 가장 명확한 주요 CTA로 배치
- 최근 학습 기록 목록
  - 학습 제목
  - 과목명
  - 생성 날짜
  - 퀴즈 정답률
- 오답노트 바로가기
- 학습 통계 카드
  - 총 학습 기록 수
  - 총 퀴즈 풀이 수
  - 평균 정답률 또는 최근 정답률
- 마이페이지 이동 아이콘 또는 버튼

5. StudyInputScreen
- 제목 입력 필드
- 과목명 입력 필드
- 학습 내용 입력 영역
  - 긴 텍스트를 붙여넣을 수 있도록 충분히 큰 입력 영역
- AI 요약 생성 버튼
- 입력 검증 상태
  - 제목이 비어 있으면 안내
  - 학습 내용이 30자 미만이면 버튼 비활성화 또는 오류 메시지 표시
- AI 요청 중 로딩 상태와 중복 클릭 방지 상태

6. SummaryResultScreen
- 원문 제목
- 과목명
- AI 요약 결과 3~5줄 표시 영역
- 핵심 키워드 3개 표시 영역
- 퀴즈 생성 버튼
- 저장 버튼 또는 저장 완료 상태
- 퀴즈 생성 실패 시 재시도 버튼과 오류 메시지 영역
- 요약 저장이 이미 완료된 경우 중복 저장을 방지하는 상태 표시

7. QuizScreen
- 현재 문제 번호와 전체 문제 수 표시 예: 1/3
- 문제 내용
- 보기 4개
- 선택된 보기 상태
- 다음 버튼
- 답을 선택하지 않으면 다음 버튼 비활성화
- 마지막 문제에서는 다음 버튼 대신 결과 보기 버튼

8. QuizResultScreen
- 전체 문제 수
- 맞힌 문제 수
- 정답률을 크게 표시
- 틀린 문제 요약 목록
- 틀린 문제 보기 버튼
- 홈으로 이동 버튼
- 사용자가 결과를 빠르게 이해할 수 있는 단순한 점수 구조

9. WrongAnswerScreen
- 틀린 문제 목록
- 문제 상세 영역
- 사용자가 선택한 답
- 정답
- 해설
- 다시 풀기 버튼
- 홈으로 돌아가기 또는 이전 화면으로 돌아가기 동선

10. MyPageScreen
- 사용자 이메일
- 총 학습 기록 수
- 총 퀴즈 풀이 수
- 로그아웃 버튼
- 앱 정보 또는 간단한 계정 정보 영역

상태 및 예외 화면도 함께 고려해라:
- 로딩 상태: AI 요약 생성 중, 퀴즈 생성 중, Firestore 데이터 조회 중
- 빈 상태: 최근 학습 기록 없음, 오답노트 없음
- 오류 상태: 로그인 실패, AI API 실패, JSON 파싱 실패, 네트워크 실패
- 입력 보존: AI 요청 실패 시 사용자가 입력한 학습 내용이 사라지면 안 된다.

디자인 방향:
- 모바일 우선 세로 화면 기준으로 설계한다.
- 학습 도구에 맞게 차분하고 명확한 정보 구조를 사용한다.
- 홈은 학습 시작 버튼과 최근 기록을 빠르게 찾을 수 있어야 한다.
- 퀴즈 화면은 문제와 보기에 집중할 수 있도록 산만한 요소를 줄인다.
- 오답노트는 반복 학습 목적이므로 정답과 해설이 한눈에 비교되어야 한다.
- 와이어프레임 단계이므로 과도한 장식보다 레이아웃, 우선순위, 상태 표현을 중점적으로 보여준다.

산출물 요구사항:
- 전체 앱 화면 흐름 다이어그램
- 화면별 모바일 와이어프레임
- 각 화면의 주요 컴포넌트 라벨
- 주요 CTA 버튼 명칭
- 로딩, 오류, 빈 상태의 위치와 표현
- 하단 탭을 사용할 경우 Home, WrongAnswer, MyPage 중심으로 제한하고, 핵심 학습 생성 흐름은 CTA 기반으로 연결
- 와이어프레임에 MVP 제외 기능을 넣지 말 것

시연용 예시 데이터:
- 제목: 데이터베이스 기본키 정리
- 과목명: 데이터베이스
- 학습 내용:
  기본키는 데이터베이스 테이블에서 각 행을 고유하게 식별하기 위해 사용하는 속성이다. 기본키 값은 중복될 수 없으며 NULL 값을 가질 수 없다. 하나의 테이블에는 기본키가 하나만 존재하며, 기본키는 개체 무결성을 유지하는 데 중요한 역할을 한다.

최종 결과는 개발자가 Flutter 화면으로 구현하기 쉬운 구조여야 하며, 각 화면의 목적과 사용자 다음 행동이 명확해야 한다.
```
