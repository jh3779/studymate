# 화면 설계서

## 1. 문서 목적

이 문서는 StudyMate의 각 화면 역할, 표시 요소, 주요 동작을 정의하여 Android Java 화면 구현의 기준으로 사용한다.

## 2. 핵심 내용

| 화면 | 역할 | 주요 요소 |
| --- | --- | --- |
| SplashScreen | 앱 시작 및 로그인 상태 확인 | 로고, 로딩 표시 |
| LoginScreen | 이메일 로그인 | 이메일, 비밀번호, 로그인 버튼, 회원가입 이동 |
| SignUpScreen | 이메일 회원가입 | 이메일, 비밀번호, 비밀번호 확인, 회원가입 버튼 |
| HomeScreen | 학습 시작과 기록 요약 | 학습 시작 버튼, 최근 기록, 오답노트, 통계 카드 |
| StudyInputScreen | 학습 내용 입력 | 제목, 과목명, 본문, 요약 생성 버튼 |
| SummaryResultScreen | AI 요약 결과 확인 | 제목, 요약, 키워드, 퀴즈 생성 버튼, 저장 버튼 |
| QuizScreen | 퀴즈 풀이 | 문제 번호, 문제, 보기 4개, 다음 버튼 |
| QuizResultScreen | 퀴즈 결과 확인 | 점수, 정답률, 맞힌 문제 수, 틀린 문제 보기 |
| WrongAnswerScreen | 오답 복습 | 틀린 문제 목록, 정답, 해설, 다시 풀기 |
| MyPageScreen | 사용자 정보 | 이메일, 학습 수, 퀴즈 수, 로그아웃 |

## 3. 상세 설명

### SplashScreen

- 앱 로고 또는 서비스명을 표시한다.
- Firebase Authentication의 현재 로그인 사용자를 확인한다.
- 로그인 상태가 있으면 HomeScreen으로 이동한다.
- 로그인 상태가 없으면 LoginScreen으로 이동한다.

### LoginScreen

- 이메일 입력 필드
- 비밀번호 입력 필드
- 로그인 버튼
- 회원가입 화면 이동 버튼
- 로그인 실패 메시지 영역

### SignUpScreen

- 이메일 입력 필드
- 비밀번호 입력 필드
- 비밀번호 확인 입력 필드
- 회원가입 버튼
- 로그인 화면으로 돌아가기 버튼

### HomeScreen

- 오늘의 학습 시작 버튼
- 최근 학습 기록 목록
- 오답노트 바로가기
- 학습 통계 카드
- 마이페이지 이동 버튼

최근 학습 기록에는 학습 제목, 과목명, 생성 날짜, 퀴즈 정답률을 표시한다.

### StudyInputScreen

- 제목 입력
- 과목명 입력
- 학습 내용 입력
- AI 요약 생성 버튼

검증 조건은 다음과 같다.

| 항목 | 조건 |
| --- | --- |
| 제목 | 비어 있으면 안 됨 |
| 학습 내용 | 최소 30자 이상 |
| 과목명 | 선택 또는 권장 입력 |

### SummaryResultScreen

- 원문 제목
- AI 요약 결과 3~5줄
- 핵심 키워드 3개
- 퀴즈 생성 버튼
- 저장 버튼

요약이 이미 저장된 경우 중복 저장을 방지해야 한다.

### QuizScreen

- 현재 문제 번호와 전체 문제 수
- 문제 내용
- 보기 1~4
- 선택 상태 표시
- 다음 버튼

답을 선택하지 않은 상태에서는 다음 버튼을 비활성화한다.

### QuizResultScreen

- 전체 문제 수
- 맞힌 문제 수
- 정답률
- 틀린 문제 목록
- 틀린 문제 보기 버튼
- 홈으로 이동 버튼

### WrongAnswerScreen

- 틀린 문제 목록
- 문제 상세 영역
- 사용자가 선택한 답
- 정답
- 해설
- 다시 풀기 버튼

### MyPageScreen

- 사용자 이메일
- 총 학습 기록 수
- 총 퀴즈 풀이 수
- 로그아웃 버튼

## 4. 개발 시 참고사항

- 화면별 입력 검증은 `utils/validators.dart`에 분리한다.
- 공통 버튼, 카드, 로딩 상태는 `res/drawable`, 공통 Java helper, 재사용 layout 기준으로 작성한다.
- AI 요청 중에는 중복 요청을 막기 위해 버튼을 비활성화한다.
- 모바일 화면 기준으로 긴 학습 내용 입력 필드는 충분한 높이를 제공한다.
